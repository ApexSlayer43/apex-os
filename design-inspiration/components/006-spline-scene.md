## What Makes This Hit

### The Core Technique — Spline
**Spline is a no-code 3D design tool that exports embeddable scenes as `.splinecode` URLs.** You design in the Spline editor (spline.design), publish, and get a URL like `https://prod.spline.design/{id}/scene.splinecode`. The `@splinetool/react-spline` package wraps the Spline runtime in a React component. You point it at the URL and it renders a fully interactive, mouse-responsive, physics-enabled 3D scene directly in the DOM.

**This is the equivalent of what UnicornStudio does in component #003** — external creative tool, embeddable scene URL, React wrapper. Spline is the 3D design version; UnicornStudio is the motion/particle effects version.

### Why Spline Hits the $10K Level
The 3D scene in the demo (a floating, animated robot/character) is built in Spline with: rigged animation, environment lighting, material shaders, physics-based idle motion, and mouse-cursor interaction that rotates the model. Getting this in Three.js would be hundreds of lines of custom code. Spline delivers it via a single `<SplineScene scene="url" />` line.

The key insight: **the quality ceiling is the Spline designer's skill, not the developer's shader knowledge.** This decouples visual complexity from engineering complexity.

### `React.lazy` + `Suspense` Pattern
Spline's runtime is heavy (~1MB+). Embedding it in the initial JS bundle would destroy First Contentful Paint. The solution:

```tsx
const Spline = lazy(() => import('@splinetool/react-spline'));
```

`React.lazy` + dynamic import means the Spline bundle is only requested when the component mounts. The `<Suspense fallback={...}>` shows a spinner while it loads. This is the correct pattern for any heavy third-party library that isn't needed on initial paint.

**The `.loader` CSS** — the fallback spinner uses a `<span className="loader">` that requires a CSS class defined in `globals.css`. It's not auto-generated by Tailwind. Add manually:
```css
.loader {
  width: 24px; height: 24px;
  border: 2px solid rgba(255,255,255,0.2);
  border-top-color: white;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
@keyframes spin { to { transform: rotate(360deg); } }
```

### Two Spotlight Variants Provided
This component ships two completely different `Spotlight` implementations:

**Aceternity variant (SVG + CSS animation):**
- A massive SVG ellipse with `feGaussianBlur` filter (stdDeviation: 151 — extremely soft)
- Positioned absolutely, sized at 138–169% of container dimensions
- `animate-spotlight` CSS animation: fires once on mount, fades from opacity:0 to 1
- Requires a Tailwind keyframe entry in `tailwind.config.js`
- Static — it doesn't track the mouse. The light is always in the same position.
- Effect: creates a dramatic one-directional stage-lighting feel. Like a single overhead spot.

**ibelick variant (Framer Motion + mouse tracking):**
- A circular `radial-gradient` div that follows the cursor in real-time
- Uses `useSpring` to add physics inertia to the position — the light lags behind the cursor slightly (`bounce: 0` = smooth follow, no overshoot)
- Mounts by mutating its parent element to add `position: relative` and `overflow: hidden` automatically
- Attaches `mousemove`/`mouseenter`/`mouseleave` listeners to the parent element
- `opacity: 0` → `opacity: 1` transition on hover
- Interactive — feels like you're physically holding a flashlight over the card

**When to use which:**
- Aceternity: static hero sections, background lighting atmosphere, "cinematic spotlight on load"
- ibelick: interactive cards, hover surfaces, anywhere mouse interaction adds engagement value

### Layout Pattern — Split Card
The demo establishes a reusable layout pattern:
```
[Card: dark bg, overflow:hidden, relative]
  [Spotlight: absolute, z-1]
  [Flex row]
    [Left 50%: text content, z-10]     ← above spotlight
    [Right 50%: 3D scene, no z-index]  ← scene renders on its own layer
```

Key decisions:
- `z-10` on the text content keeps it above the spotlight SVG (`z-1`) and readable
- `overflow-hidden` on the Card clips both the spotlight SVG (which is 138% wide) and the 3D scene at the card boundary
- `bg-black/[0.96]` — 96% opacity black. Not pure black — the 4% transparency is intentional for subtle depth/layering effects
- `bg-gradient-to-b from-neutral-50 to-neutral-400` on the h1 — "ghost gradient text" pattern. Headline appears slightly lighter at top, darker at bottom, giving it a 3D-illuminated look on dark backgrounds

### The Gradient Text Technique
```tsx
className="bg-clip-text text-transparent bg-gradient-to-b from-neutral-50 to-neutral-400"
```
Three classes working together:
1. `bg-gradient-to-b from-neutral-50 to-neutral-400` — sets the gradient as a background
2. `bg-clip-text` — clips that background to only show through the text glyphs
3. `text-transparent` — makes the text color transparent so the background shows through

Result: text that appears to have a vertical gradient painted on it. Standard technique for premium dark-theme headlines. Works with any Tailwind gradient direction/color.

### Design Principles to Extract
1. **Spline = Three.js without the engineering** — for complex 3D scenes on marketing/hero pages, Spline is the correct tool. Reserve custom Three.js for truly bespoke interactivity that Spline can't export.
2. **`React.lazy` + `Suspense` is mandatory for heavy 3D/canvas libraries** — never import Spline, Three.js, or similar synchronously. Bundle size → LCP → conversions.
3. **Two spotlight variants serve two use cases** — static spotlights for cinematic atmosphere; mouse-tracking spotlights for interactive engagement. Know which you need before implementing.
4. **`bg-clip-text` + `text-transparent` + gradient** is the fastest way to elevate dark-theme headline typography. 3 classes, zero additional images.
5. **`bg-black/[0.96]` not `bg-black`** — pure black is flat. Near-black with slight transparency allows underlying colors/gradients to breathe through, creating subtle depth.
6. **Splitting a card 50/50 content|media** is the foundational "product showcase" layout. Left = what it is + why it matters. Right = what it looks like.
7. **Auto-mutating parent in spotlight** (ibelick variant) — setting `position: relative` and `overflow: hidden` on the parent via `useEffect` is a smart self-configuring pattern. The component doesn't require wrapper setup from the consumer.
8. **`useSpring` for cursor-following effects** — `bounce: 0` gives zero overshoot, smooth lag. This is the only correct way to do cursor follower effects. Direct position tracking looks mechanical; spring physics looks alive.
9. **The Spline scene URL is the integration point** — content teams/designers can swap 3D scenes by just changing the URL. No code change needed. This separates design from engineering cleanly.
10. **`feGaussianBlur stdDeviation="151"` = extreme softness** — this is not a subtle blur. 151px blur radius on a massive SVG ellipse creates a massive soft glow. For a "single overhead light" effect, you need blur radius proportional to the light size.
